# Project: ODTrack Academia™

## 1. Overview

ODTrack Academia™ is a comprehensive, production-ready On-Duty (OD) request management system designed for academic institutions. Built with Flutter and FastAPI, it provides a seamless, offline-capable experience for both students and staff across multiple platforms (Android, iOS, Web, Windows, macOS, Linux). The application is architected to be robust, scalable, and to work efficiently even in offline scenarios with automatic synchronization when connectivity is restored.

## 2. Core Technologies

## 3. Key Features

### 3.1. Authentication & User Roles

- **Dual Login System:** Separate authentication flows for students (using Register Number and Date of Birth) and staff (using Email and Password)
- **Secure Token Management:** JWT tokens stored with AES-256 encryption using flutter_secure_storage
- **Role-Based Access Control:** Different dashboards and functionalities based on user role (student or staff)
- **Session Management:** Automatic session handling with logout functionality

### 3.2. Student Features

- **Dashboard:** Personalized dashboard with quick stats on OD requests (pending, approved, rejected) and quick actions
- **New OD Request:** Intuitive form for submitting OD requests with:
  - Date and period selection
  - Automatic staff member assignment based on timetable
  - Optional Class and Year Coordinator sections with dual selection confirmation
  - 30-second undo buffer after submission for safety
  - Toast notifications for staff information
- **OD Request Management:** View, track, and manage submitted OD requests with:
  - Real-time status tracking
  - Request history with detailed information
  - Status change notifications
- **Timetable:** View class timetables for different years and sections with color-coded subjects
- **Staff Directory:** Browse directory of all staff members with:
  - Contact details and subject information
  - Search functionality
  - Department filtering
  - Direct access to staff profiles
- **Push Notifications:** Real-time alerts for OD request status changes
- **Calendar Integration:** Automatically sync approved OD requests with device calendar
- **Offline Support:** Full app functionality without internet connection with automatic sync when online

### 3.3. Staff Features

- **Dashboard:** Staff-centric dashboard with stats on pending requests, total requests, and quick actions
- **OD Inbox:** Dedicated inbox for reviewing and managing OD requests with:
  - Filter by status (All/Pending/Approved/Rejected)
  - Approve/reject with confirmation dialogs
  - Reason tracking for rejections
  - Real-time status updates with visual feedback
  - Bulk operations for efficient batch processing
- **Personal Timetable:** View personal weekly teaching schedules with color-coded subjects
- **Staff Profile:** View and edit personal profile information with:
  - Contact details management
  - Password change functionality
  - Quick access to help and support
- **Push Notifications:** Real-time alerts for new OD requests and status updates
- **Bulk Operations:** Efficiently manage multiple OD requests with:
  - Multi-select UI with checkboxes
  - Batch approval, rejection, and export
  - Real-time progress tracking
  - Error handling and reporting
  - Undo functionality with time window
- **Staff Analytics & Workload Management:** Detailed insights into:
  - Workload tracking with weekly/monthly breakdown
  - Teaching analytics with subject-wise period allocation
  - Time allocation tracking by activity type
  - Efficiency metrics and performance tracking
  - Comparative analytics (semester-over-semester)
  - Department benchmarking and peer comparison
  - Proactive alerts for workload imbalances
  - Dark mode support for all components
- **Calendar Integration:** Sync approved OD requests with device calendar
- **PDF Export:** Generate and share PDF reports for various data

### 3.4. Analytics & Reporting

- **Analytics Dashboard:** Powerful analytics dashboard for staff with visualizations using fl_chart:
  - **Request Status Distribution:** Pie chart showing proportion of pending, approved, and rejected requests
  - **Monthly Request Volume:** Bar chart tracking volume of OD requests over time
  - **Top Rejection Reasons:** Pie chart identifying common reasons for rejections
  - **Request Trends:** Line chart analyzing trends in OD requests
  - **Department-wise Comparison:** Bar chart comparing OD request volumes across departments
- **Staff Analytics & Workload Management:** Comprehensive tracking and analysis of staff performance:
  - **Workload Analytics:** Track working hours per semester with weekly/monthly breakdown
  - **Teaching Analytics:** Subject-wise period allocation and grade-wise teaching distribution
  - **Time Allocation Tracking:** Daily/weekly schedule breakdown by activity type
  - **Efficiency Metrics:** OD processing performance and comparative benchmarking
  - **Comparative Analytics:** Semester-over-semester performance comparison
  - **Department Benchmarking:** Comparison with department and institutional averages
- **Data Export:** Export analytics reports in PDF and CSV formats with customization options

### 3.5. Offline Support & Synchronization

- **Robust Offline Functionality:** Application is fully functional without internet connection
- **Sync Queue:** All operations queued locally using SyncQueueManager when offline
- **Background Sync Worker:** Automatically synchronizes queued data when connectivity restored with:
  - **Connectivity Monitoring:** Detects changes in network connectivity to trigger sync
  - **Exponential Backoff:** Retry mechanism with exponential backoff for failed sync attempts
  - **Periodic Sync:** Configurable periodic synchronization intervals
- **Conflict Resolution:** Mechanism to handle data conflicts during synchronization
- **Intelligent Cache Management:** Efficient caching with TTL strategies and priority-based cleanup

### 3.6. Intelligent Caching

- **Efficient Data Caching:** IntelligentCacheManager caches frequently accessed data locally
- **Time-to-Live (TTL) Strategies:** Different TTL strategies for various data categories:
  - User profiles: Longer TTL
  - OD requests: Medium TTL
  - Timetables: Longer TTL
  - Analytics data: Shorter TTL
- **Priority-Based Cleanup:** Cache managed efficiently with priority-based cleanup mechanism
- **Memory Management:** Automatic eviction of less important data when cache is full
